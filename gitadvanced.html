Advanced Git Concepts

Rebasing

Rebasing rewrites commit history by integrating changes from one branch into another.


# Switch to the feature branch
git checkout feature-branch

# Rebase onto the main branch
git rebase main

After rebasing, resolve conflicts (if any) and continue:


# Resolve conflicts manually
git add 

# Continue rebasing
git rebase --continue

Stashing

Stashing allows you to temporarily save uncommitted changes.


# Save changes to a stash
git stash

# List stashes
git stash list

# Apply the latest stash
git stash apply

To drop or clear a stash:


# Drop the latest stash
git stash drop

# Clear all stashes
git stash clear

Cherry-Picking

Cherry-picking lets you apply a specific commit from one branch to another.


# Apply a specific commit (use the commit hash)
git cherry-pick

Squashing Commits

Squashing merges multiple commits into one, simplifying the commit history.


# Interactive rebase to squash commits
git rebase -i HEAD~3

# Mark commits as 'squash' (or 's') in the interactive editor
# Save and exit

Git Bisect

Bisect helps to find a commit that introduced a bug using binary search.


# Start bisecting
git bisect start

# Mark the current commit as bad
git bisect bad

# Mark a known good commit
git bisect good 

# Git will checkout commits for testing. Mark each as good or bad until the culprit is found.
