Advanced Git Concepts

Stashing

Stashing allows you to save changes without committing them, so you can switch branches or work on something else.


# Save changes to the stash
git stash

# List stashed changes
git stash list

# Apply the latest stash
git stash apply

# Apply and remove the latest stash
git stash pop

Rebasing

Rebasing replays commits from one branch onto another, creating a linear commit history.


# Rebase feature branch onto main
git checkout feature-branch
git rebase main

# Resolving conflicts during rebase
# Edit conflicting files, then run:
git rebase --continue

Cherry-Picking

Cherry-picking allows you to apply a specific commit from one branch to another.


# Cherry-pick a specific commit
git cherry-pick 

# Example
git cherry-pick a1b2c3d4

Working with Tags

Tags mark specific points in your repository's history, often used for versioning.


# Create an annotated tag
git tag -a v1.0 -m "Version 1.0"

# List all tags
git tag

# Push tags to remote
git push origin --tags
