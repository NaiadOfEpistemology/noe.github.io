Introduction to C Language

History of C

The C programming language was developed by Dennis Ritchie at Bell Labs in the early 1970s. It was designed as a system programming language for writing operating systems. The development of C was influenced by earlier languages such as B and BCPL. C gained widespread popularity because of its efficiency, portability, and the richness of its standard library. It has influenced many subsequent programming languages, including C++, Java, and Python.

Applications

C is used in a wide variety of applications, including:

  • Operating Systems (e.g., Unix, Linux, Windows)
  • Embedded Systems
  • System Programming
  • Compiler Design
  • Game Development
  • Real-time Systems
  • Database Systems

Features

C is known for several key features:

  • Simplicity: C has a straightforward syntax and a small set of keywords.
  • Efficiency: C provides low-level access to memory and hardware, making it a powerful language for system programming.
  • Portability: Programs written in C can be easily adapted to different computer systems.
  • Modularity: C supports functions and libraries, allowing code to be reused and organized.
  • Rich Library: The C standard library provides numerous built-in functions for common tasks.
  • Flexibility: C allows direct manipulation of hardware and memory, making it suitable for low-level programming.

Structure of a C Program

A typical C program consists of the following components:

  • Preprocessor Directives: Instructions to the compiler to process before compilation (e.g., #include).
  • Main Function: The entry point of the program where execution begins (e.g., int main()).
  • Variable Declarations: Declaring variables used in the program.
  • Statements and Expressions: Instructions to be executed by the program.
  • Functions: Blocks of code that perform specific tasks and can be reused.

#include <stdio.h>  // Preprocessor directive

// Main function
int main() {
    // Variable declaration
    int num = 10;

    // Print statement
    printf("Hello, World! Number: %d\n", num);

    // Return statement
    return 0;
}

Execution Flow of a C Program

The execution flow of a C program typically follows these steps:

  • Preprocessing: The preprocessor processes the preprocessor directives.
  • Compilation: The compiler translates the C code into assembly code.
  • Assembly: The assembler converts the assembly code into machine code.
  • Linking: The linker combines the machine code with necessary libraries to create an executable file.
  • Execution: The executable file is loaded into memory and executed by the operating system.
